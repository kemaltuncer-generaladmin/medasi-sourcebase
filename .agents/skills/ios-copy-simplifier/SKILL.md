# iOS Copy Simplifier

## Core Stance

Make the app say less and help more. Prefer short, human, action-oriented text. Remove explanations that exist because the UI is unclear; fix the UI or label first.

Avoid explaining the app too much. Users should understand what to do from titles, labels, placement, and sensible defaults.

## Review Scope

Review button labels, tab labels, empty states, onboarding text, error messages, alerts, permission explanations, settings labels, helper text, placeholders, validation text, sheet titles, banners, badges, and toast messages.

## Rewrite Rules

- Prefer direct verbs: "Create", "Retry", "Continue", "Save", "Delete", "Open".
- Replace technical or system-centered copy with user-centered outcomes.
- Delete repeated text where the title, button, or surrounding UI already says it.
- Keep alerts short: what happened, what the user can do next.
- Keep permission copy honest and specific.
- Avoid vague praise, filler reassurance, marketing language, and internal feature names unless the user needs them.
- Use consistent terms for the same object across screens.
- Preserve clinical, legal, payment, privacy, and safety meaning when shortening text.

## Output Format

- Original copy
- Problem
- Suggested shorter copy
- Reason
- Screens affected
