### Headline gaps found (evidence)

1. **Dynamic Type is broken in ~194 places** — `grep` finds 194 `.font(.system(size:` fixed-point fonts across the module (icons, badges, tab labels, chevrons). These do **not** scale with the user's text size. `SBTypography` itself is correct (uses `Font.system(.headline)` text styles), but most call sites bypass it.
2. **No iPad / size-class adaptation** — `grep` for `horizontalSizeClass`, `sizeClass`, `UIDevice`, `idiom` returns **zero** matches. Layout relies only on `.frame(maxWidth: 1180)` clamps and `LazyVGrid(.adaptive(minimum:))`. No regular-width split or column-count changes.
3. **Reduce Motion is almost entirely ignored** — only **1** file (`DesignSystem/SBPremiumVisuals.swift:164`) reads `@Environment(\.accessibilityReduceMotion)`. The global `sbEntrance` stagger (`SBMotion.swift`), `matchedGeometryEffect` tab pill, Pow `.bounce`/`.shine`, and `symbolEffect(.bounce)` all animate unconditionally. The `sbEntrance` doc comment *claims* Reduce Motion is auto-handled — it is not.
4. **Non-lazy scrolling content** — 37 files use `ScrollView { VStack { ForEach … } }`. Long lists (`UploadsView`, `QueueView`, `CollectionsView`, `SearchView`, `FolderView`, `ProfileView`) render every row eagerly. Only 19 files use `LazyVStack`/`LazyVGrid`.
5. **Incomplete VoiceOver grouping** — interactive cards (`SBFileCard`, `collectionCard`, `courseRow`, `storeSpotlightSection`) are `Button`s with rich child text but **no** `.accessibilityElement(children: .combine)` + `.accessibilityAddTraits(.isButton)`, so VoiceOver reads each fragment separately and omits the action hint. Only 49 accessibility modifiers exist across the whole module.
6. **Dark mode is unverified / likely wrong** — `SBColors` are fixed light-mode RGB literals (e.g. `white = Color.white`, `navy` near-black text). `preferredColorScheme` is wired from a preference, but there are no adaptive/asset colors, so dark mode would show dark-on-dark or light cards on a dark system background.
7. **Identity / re-render hygiene** — `ForEach(Array(workspace.courses.prefix(3)), id: \.id)` and `id: \.file.id` are fine, but no view conforms to `Equatable` and `@Observable` stores expose broad arrays; mutating one file re-evaluates entire home screens.

## Proposed Code Changes

### 1. Reduce-Motion-aware `SBEntranceModifier` — `DesignSystem/SBMotion.swift`

```swift
private struct SBEntranceModifier: ViewModifier {
    let index: Int
    @Environment(\.accessibilityReduceMotion) private var reduceMotion
    @State private var appeared = false

    func body(content: Content) -> some View {
        content
            .opacity(appeared ? 1 : 0)
            .offset(y: appeared ? 0 : (reduceMotion ? 0 : 14))
            .scaleEffect(appeared ? 1 : (reduceMotion ? 1 : 0.985), anchor: .top)
            .onAppear {
                guard !appeared else { return }
                if reduceMotion {
                    appeared = true            // no spring, no transform
                } else {
                    withAnimation(SBMotion.softSpring.delay(SBMotion.stagger(index))) {
                        appeared = true
                    }
                }
            }
    }
}
```
